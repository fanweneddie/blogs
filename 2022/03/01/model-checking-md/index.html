Introduction to Model Checking

Created2022-03-01Updated2022-03-01

This is a brief introduction to simple model checking algorithm, which refers to “Logic in Computer Science”, “Model Checking” and “Handbook of Model Checking”.

Introduction

Formal verification can conduct an exhaustive exploration of all possible behaviors, and one of its approaches is model checking, which is automatic on success and can produce a counterexample on failure.

Basically, model checking requires 3 steps:

  • Modeling: To convert a design into a formalism accepted by a model checking tool
  • Specification: To state the properties that the design should satisfy
  • Verification: To check whether the design satisfies the specification

Problem

Given a Kripke structure M=(S,R,L)M = (S, R, L)M=(S,R,L) that represents a finite state system and a temporal logic formula fff that expresses the specification, we need to find the set S1S_1S1​ of all states in SSS s.t.

S1={s∈S∣M,s⊨f}(1)S_1 = \{ s \in S \mid M, s \vDash f \} \tag{1} S1​={s∈S∣M,s⊨f}(1)

There are many types of representation of Kripke structure. We just use the easiest one: A labeled, directed graph with arcs. Here, each node represents a state sss in SSS, and each arc represents a transition relation rrr in RRR, and each label is a function L:S→2APL: S \rightarrow 2^{AP}L:S→2AP that gets the true formulas on sss.

Searching the states that satisfy a LTL formula fff cannot leverage the labelling approach above, since the subformulas of a LTL formula should be evaluated along paths (starting from them) instead of in their states.
