Symbolic Automata-based String Analysis

Created2022-03-15Updated2022-03-15

This is the note of Automata Based String Analysis.

Overview

Given an input sanitizer program, we can build a dependency graph for strings and their operation. On that dependency graph, we can first do forward analysis to propagate string information. When a sink function is reached, we intersect its states with attack pattern. If the intersection is not empty, we do a backward analysis to propagate the result backwards to identify which inputs can cause an attack.

At sink node, we intersect the DFA for the sink nodes with the DFA for the attack pattern to identify vulnerabilities. In Figure 1-3, we find that the intersection is not empty, showing that the vulnerability may occur.

Automata Lattice

Given a deterministic finite automaton (DFA) AAA, let L(A)L(A)L(A) denote the set of values accepted by AAA. We define A1⊑A2A_1 \sqsubseteq A_2A1​⊑A2​ iff L(A1)⊆L(A2)L(A_1) \subseteq L(A_2)L(A1​)⊆L(A2​). We define A1⊔A2A_1 \sqcup A_2A1​⊔A2​ as a DFA s.t. L(A1⊔A2)=L(A1)∪L(A2)L(A_1 \sqcup A_2) = L(A_1 ) \cup L(A_2)L(A1​⊔A2​)=L(A1​)∪L(A2​) and A1⊓A2A_1 \sqcap A_2A1​⊓A2​ as a DFA s.t. L(A1⊓A2)=L(A1)∩L(A2)L(A_1 \sqcap A_2) = L(A_1 ) \cap L(A_2)L(A1​⊓A2​)=L(A1​)∩L(A2​).

Given a regular expression rrr, we use A(r)A(r)A(r) to denote an automaton that accepts the set of strings that match rrr. Similarly, given a set of strings P⊆Σ∗P \subseteq \Sigma^*P⊆Σ∗, we use A(P)A(P)A(P) to denote an automaton where L(A(P))=PL(A(P)) = PL(A(P))=P.

We can define an automata lattice, where

  • each element is an automaton that accepts a set of strings,
  • with the partial order, join operator and meet operator defined above,
  • with top element ⊤=A(Σ∗)\top = A(\Sigma^*)⊤=A(Σ∗) and bottom element ⊥=A(∅)\bot = A(\empty)⊥=A(∅).

Note that this lattice is an infinite lattice. For example, we have an infinite chain A(∅)⊑A({ϵ})⊑A({ϵ,a})⊑A({ϵ,a,aa})...⊑A(a∗)A(\empty) \sqsubseteq A(\{ \epsilon \}) \sqsubseteq A(\{ \epsilon, a \}) \sqsubseteq A(\{ \epsilon, a, aa \}) ... \sqsubseteq A(a^*)A(∅)⊑A({ϵ})⊑A({ϵ,a})⊑A({ϵ,a,aa})...⊑A(a∗). In order to make it converge during fixpoint computation, we need to use a widening operator ▽\triangledown▽ to compute an over-approximation of the join operator, i.e. A1⊔A2⊑A1▽A2A_1 \sqcup A_2 \sqsubseteq A_1 \triangledown A_2A1​⊔A2​⊑A1​▽A2​.

Symbolic Automata Representation

  • QQQ is a finite set of states,
  • q0q_0q0​ is the initial state,
  • ΣB\Sigma_BΣB​ is a symbolic binary alphabet,
  • F⊆QF \subseteq QF⊆Q is a finite set of accepting states,
  • δ:Q×ΣB→Q\delta: Q \times \Sigma_B \rightarrow Qδ:Q×ΣB​→Q is the transition relation.

Widening

In order to reach a fixpoint faster, we need to compute a sound approximation of the least fixpoint that corresponds to the reachable states.

Given two finite automata M=<Q,q0,Σ,δ,F>M = <Q, q_0, \Sigma, \delta, F>M=<Q,q0​,Σ,δ,F> and M′=<Q′,q0′,Σ,δ′,F′>M'= <Q', q_0', \Sigma, \delta', F'>M′=<Q′,q0′​,Σ,δ′,F′>, we can first define binary relation ≡▽\equiv_{\triangledown}≡▽​ on Q∪Q′Q\cup Q'Q∪Q′.
