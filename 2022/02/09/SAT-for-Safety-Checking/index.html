SAT for Safety Checking

Created2022-02-09Updated2022-02-09

This is the note of paper “Checking safety properties using induction and a SAT-solver” by Mary Sheeran et al. at FMCAD 2000.

Problem

The authors are interested in the problem of checking safety properties of large finite state machines using a SAT-solver. Given a finite state machine MMM with initial states satisfying III and state transition relation TTT, we would like to check whether or not a property PPP holds for all reachable states.

Definitions

We write T(x,y)T(x,y)T(x,y) to indicate that xxx is related to yyy by the transition relation TTT. A sequence of states through TTT can be defined as a path:

path(s[0..n])=∧0≤i<nT(si,si+1)(1)path(s_{[0..n]}) = \land_{0 \leq i < n} T(s_i,s_{i+1}) \tag{1} path(s[0..n]​)=∧0≤i<n​T(si​,si+1​)(1)

where s[0..n]s_{[0..n]}s[0..n]​ is shorthand for the sequence of state (s0,s1,...,sn)(s_0,s_1,...,s_n)(s0​,s1​,...,sn​). To assert that a property QQQ holds at every point in a path, we write all.Q(s[0..n])all.Q(s_{[0..n]})all.Q(s[0..n]​).

We can view path as a binary relation on points, so we write pathi(s0,si)path_i(s_0, s_i)pathi​(s0​,si​) to indicate that there is a path from s0s_0s0​ to sis_isi​ through iii copies of TTT.

pathn(s0,sn)↔∃s1...sn−1 path(s[0...n])(2)path_n(s_0, s_n) \leftrightarrow \exist s_1...s_{n-1}\ path(s_{[0...n]}) \tag{2} pathn​(s0​,sn​)↔∃s1​...sn−1​ path(s[0...n]​)(2)

If every element of a path is distinct, then we define it as loop-free:

loopFree(s[0..n])=path(s[0..n])∧(∧0≤i<j≤nsi≠sj)(3)loopFree(s_{[0..n]}) = path(s_{[0..n]}) \land (\land_{0 \leq i < j \leq n} s_i \neq s_j) \tag{3} loopFree(s[0..n]​)=path(s[0..n]​)∧(∧0≤i<j≤n​si​​=sj​)(3)

Finally, a path from aaa to bbb is shortest if it joins aaa and bbb and if aaa and bbb are not joined by any shorter path. That is,

shortest(s[0..n])=path(s[0..n])∧¬(∨0≤i<n pathi(s0,sn))(4)shortest(s_{[0..n]}) = path(s{[0..n]}) \land \neg(\lor_{0 \leq i < n} \ path_i(s_0,s_n)) \tag{4} shortest(s[0..n]​)=path(s[0..n])∧¬(∨0≤i<n​ pathi​(s0​,sn​))(4)

Formulating the problem

Given a set of states SSS and its transition relation TTT (assuming that the domain of TTT is the entire set of states SSS), the property III of initial states, we want to check that, starting in an initial state and repeatedly applying the transition relation always leads to a state satisfying PPP.

∀i.∀s0...si.(I(s0)∧path(s[0...i])→P(si))(5)\forall i. \forall s_0...s_i. (I(s_0) \land path(s_{[0...i]}) \rightarrow P(s_i)) \tag{5} ∀i.∀s0​...si​.(I(s0​)∧path(s[0...i]​)→P(si​))(5)

Or we can go backwards from the bad states: we can check whether starting in a state violating PPP and working backwards through TTT leads to a non-initial state,

∀i.∀s0...si.(path(s[0...i])∧¬P(si)→¬I(s0))(6)\forall i. \forall s_0...s_i. (path(s_{[0...i]}) \land \neg P(s_i) \rightarrow \neg I(s_0)) \tag{6} ∀i.∀s0​...si​.(path(s[0...i]​)∧¬P(si​)→¬I(s0​))(6)

Both of these can be the same as proving

∀i.∀s0...si.¬(I(s0)∧path(s[0...i])∧¬P(si))(7)\forall i. \forall s_0...s_i. \neg (I(s_0) \land path(s_{[0...i]}) \land \neg P(s_i)) \tag{7} ∀i.∀s0​...si​.¬(I(s0​)∧path(s[0...i]​)∧¬P(si​))(7)

Solution

Solution 1

We want to divide the problem into smaller sub-problems.

Here, we check whether formula (7)(7)(7) holds for i=0,1,2i = 0,1,2i=0,1,2 and so on, which corresponds to checking that

I(s0)∧path(s[0...i])∧¬P(si)(8)I(s_0) \land path(s_{[0...i]}) \land \neg P(s_i) \tag{8} I(s0​)∧path(s[0...i]​)∧¬P(si​)(8)

is contradictory for each iii and arbitrary s0s_0s0​ to sis_isi​. If we have found the first iii that makes (8)(8)(8) satisfiable, then we know that there is a path of length iii (and there is no shorter one) from an initial state to one violating PPP.

Idea

If the system is PPP-safe, formula (7)(7)(7) will always hold. But how do we know when it is safe to stop incrementing iii and conclude that the system is PPP-safe?

It is not a good idea to wait for

I(s0)∧path(s[0..i])(9)I(s_0) \land path(s_{[0..i]}) \tag{9} I(s0​)∧path(s[0..i]​)(9)

to be contradictory, since there are infinite loops to check (note that the state transfer graph is a directed cyclic graph).

However, we can stop when

I(s0)∧loopFree(s[0...i])(10)I(s_0) \land loopFree(s_{[0...i]}) \tag{10} I(s0​)∧loopFree(s[0...i]​)(10)

becomes contradictory. Then, we stop when we have checked every loop-free path (and thus every state) in the reachable states. We can then safely conclude that the system is PPP-safe.

Similarly, we can keep checking until

loopFree(s[0..i])∧¬P(si)(11)loopFree(s_{[0..i]}) \land \neg P(s_i) \tag{11} loopFree(s[0..i]​)∧¬P(si​)(11)

becomes contradictory, when we have checked all states reachable backwards from those violating PPP.

Therefore, this algorithm is shown in Figure 1 below. Here, the function SatSatSat takes an expression and returns true if there exists an assignment to the variables s[0..i]s_{[0..i]}s[0..i]​ which make the whole expression true, and the trace c[0..i]c_{[0..i]}c[0..i]​ is an assignment to s[0..i]s_{[0..i]}s[0..i]​ that makes (8)(8)(8) true.

Figure 1: Algorithm 1

This algorithm can be thought of as working both forwards from the initial state and backwards from the bad states at the same time.

Sound & Complete

When the answer is true, from the first if statement, there exists a smallest kkk s.t. I(s0)∧loopFree(s[0..k])I(s_0) \land loopFree(s_{[0..k]})I(s0​)∧loopFree(s[0..k]​) or loopFree(s[0..k])∧¬P(sk)loopFree(s_{[0..k]}) \land \neg P(s_k)loopFree(s[0..k]​)∧¬P(sk​) is contradictory. Also, from the second if statement, for i<ki < ki<k, I(s0)∧path(s[0..i])∧¬P(si)I(s_0) \land path(s_{[0..i]}) \land \neg P(s_i)I(s0​)∧path(s[0..i]​)∧¬P(si​) is contradictory. Hence, for i<ki < ki<k,

I(s0)∧loopFree(s[0..i])∧¬P(si)(12)I(s_0) \land loopFree(s_{[0..i]}) \land \neg P(s_i) \tag{12} I(s0​)∧loopFree(s[0..i]​)∧¬P(si​)(12)

is also contradictory.

For i≥ki \geq ki≥k, it is obvious that

I(s0)∧loopFree(s[0..i])∧¬P(si)→I(s0)∧loopFree(s[0..k])∧loopFree(s[k..i])∧¬P(si)(13)I(s_0) \land loopFree(s_{[0..i]}) \land \neg P(s_i) \rightarrow I(s_0) \land loopFree(s_{[0..k]}) \land loopFree(s_{[k..i]}) \land \neg P(s_i) \tag{13} I(s0​)∧loopFree(s[0..i]​)∧¬P(si​)→I(s0​)∧loopFree(s[0..k]​)∧loopFree(s[k..i]​)∧¬P(si​)(13)

Note that I(s0)∧loopFree(s[0..k])I(s_0) \land loopFree(s_{[0..k]})I(s0​)∧loopFree(s[0..k]​) or loopFree(s[0..k])∧¬P(sk)loopFree(s_{[0..k]}) \land \neg P(s_k)loopFree(s[0..k]​)∧¬P(sk​) is contradictory. If ¬P(sk)\neg P(s_k)¬P(sk​) is contradictory, then property PPP is obviously globally valid; Otherwise, we can see that the right hand side of (13)(13)(13) is contradictory. Hence, for i≥ki \geq ki≥k, formula (12)(12)(12) is contradictory.

So we conclude that for all iii, formula (12)(12)(12) is contradictory. Thus, if the answer is true, then there is no loop-free path from the initial state to the final state that violates PPP, and the system must be PPP-safe. Therefore, the method is sound.

Also, if the system is PPP-safe, it must return true, otherwise it will return an error trace. Therefore, the algorithm is complete.

Solution 2.1

We try to improve solution 1 by making the two termination conditions a bit tighter.

Idea

In the forward direction, we don’t want to go back into an initial state as there would be a longer path than necessary. Therefore, we change the termination condition (10)(10)(10) to

I(s0)∧all.¬I(s[1..i])∧loopFree(s[0..i])(14)I(s_0) \land all.\neg I(s_{[1..i]}) \land loopFree(s_{[0..i]}) \tag{14} I(s0​)∧all.¬I(s[1..i]​)∧loopFree(s[0..i]​)(14)

Similarly, in the backwards direction, we are not interested in paths that have a non-PPP-state somewhere in the middle, and we just consider paths in which only the last state satisfies PPP. Therefore, we change the termination condition (11)(11)(11) to

loopFree(s[0..i])∧all.P(s[0..(i−1)])∧¬P(si)(15)loopFree(s_{[0..i]}) \land all.P(s_{[0..(i-1)]}) \land \neg P(s_i) \tag{15} loopFree(s[0..i]​)∧all.P(s[0..(i−1)]​)∧¬P(si​)(15)

Solution 2.2

The algorithms in Solution 2.1 is unsatisfactory when we consider the necessary number of iterations before it terminates. The number of iteration should be the shorter one between the length of the longest loop-free path from an initial state and the length of longest loop-free path consisting pf all PPP states followed by a non-PPP-state.
