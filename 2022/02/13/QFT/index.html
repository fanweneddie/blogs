Quantum Fourier Transform

Created2022-02-13Updated2022-02-14

This note is about the basic knowledge of Quantum Fourier Transform (QFT), and it refers to “Algorithms”, “Quantum Computation and Quantum Information”, wikipedia and Qiskit.

Quantum Algorithm

We can store bits in hydrogen atom. Since the single electron in an atom can be in the ground state ∣0⟩|0 \rangle∣0⟩ or in the excited state ∣1⟩|1 \rangle∣1⟩, we can use these two states to encode for bit values 000 and 111, respectively.

The superposition principle is counterintuitive: if a quantum system can be in one of two states, then it can also be in any linear superposition of those two states. For example, the state of electron could be α0∣0⟩+α1∣1⟩\alpha_0 |0\rangle + \alpha_1 |1 \rangleα0​∣0⟩+α1​∣1⟩, where the coefficient α0\alpha_0α0​ and α1\alpha_1α1​ are amplitude. Also, those complex amplitudes are normalized as ∣α0∣2+∣α1∣2=1| \alpha_0|^2 + |\alpha_1|^2 = 1∣α0​∣2+∣α1​∣2=1. Such a superposition, α0∣0⟩+α1∣1⟩\alpha_0 |0\rangle + \alpha_1 |1\rangleα0​∣0⟩+α1​∣1⟩, is the basic unit of encoded information in quantum computers and it is called a qubit.

The linear superposition, is the private world of the electron. If we want to get a glimpse of the electron’s state, we must make a measurement. If the state of the electron is ∣α0∣2+∣α1∣2=1| \alpha_0|^2 + |\alpha_1|^2 = 1∣α0​∣2+∣α1​∣2=1, then the outcome of the measurement is 000 with probability ∣α0∣2|\alpha_0|^2∣α0​∣2 and 111 with probability ∣α1∣2|\alpha_1|^2∣α1​∣2. Also, the new state is disturbed by the measurement outcome.

Quantum algorithm takes the input of nnn classical bits as nnn qubits in state ∣x⟩|x \rangle∣x⟩. After a series of quantum operations, the state of the nnn qubits has been transformed to some superposition ∑yαy∣y⟩\sum_{y} \alpha_{y} | y \rangle∑y​αy​∣y⟩. At last, a measurement is made, and the output is the nnn bit string yyy with probability ∣αy∣2|\alpha_y|^2∣αy​∣2.

Fourier Transform

The discrete Fourier Transform maps the vector (x0,x1,...,xN−1)(x_0,x_1,...,x_{N-1})(x0​,x1​,...,xN−1​) into the vector (y0,y1,...,yN−1)(y_0,y_1,...,y_{N-1})(y0​,y1​,...,yN−1​), according to the formula

yk=1N∑j=0N−1xjwNjk(1)y_k = \frac{1}{\sqrt{N}} \sum_{j=0}^{N-1} x_j w_{N}^{jk} \tag{1} yk​=N​1​j=0∑N−1​xj​wNjk​(1)

If we pick www randomly by definition, Fourier Transform requires O(N2)O(N^2)O(N2) time. However, we can choose wNjk=e2πijkNw_{N}^{jk} = e^{2\pi i\frac{jk}{N}}wNjk​=e2πiNjk​ and do recursion in a Fast Fourier Transform (FFT), which only requires O(Nlg⁡N)O(N \lg{N})O(NlgN) time. Here, we set www as an NNN-th root of unity in order to

  1. make it possible to Divide&Conquer by odd- and even-degree polynomials due to the periodicity,

  2. make the inversion matrix MN(w)−1=1NMN(w−1)M_N(w)^{-1} = \frac{1}{N}M_N(w^{-1})MN​(w)−1=N1​MN​(w−1) to make the inverse FFT (IFFT) also O(Nlg⁡N)O(N \lg{N})O(NlgN).

Quantum Fourier Transform

Idea

Similar to the classical Fourier Transform, the quantum Fourier Transform maps a quantum state ∣X⟩=∑j=0N−1xj∣j⟩|X \rangle = \sum_{j=0}^{N-1} x_j |j \rangle∣X⟩=∑j=0N−1​xj​∣j⟩ into the quantum state ∣Y⟩=∑k=0N−1xk∣k⟩|Y \rangle = \sum_{k=0}^{N-1} x_k |k \rangle∣Y⟩=∑k=0N−1​xk​∣k⟩, according to the formula (1)(1)(1) or

∣j⟩↦1N∑k=0N−1wNjk∣k⟩(2)|j \rangle \mapsto \frac{1}{\sqrt{N}} \sum_{k=0}^{N-1} w_{N}^{jk} |k \rangle \tag{2} ∣j⟩↦N​1​k=0∑N−1​wNjk​∣k⟩(2)

QFT transforms from computational (Z) basis to the Fourier basis.

QFT(∣x⟩)=∣x~⟩(3)QFT(|x \rangle) = |\tilde{x}\rangle \tag{3} QFT(∣x⟩)=∣x~⟩(3)

In the Z basis, we store numbers in binary using the states ∣0⟩|0 \rangle∣0⟩ and ∣1⟩|1\rangle∣1⟩ (as is shown in Figure 2).

In the Fourier basis, we store numbers using different rotations around the Z-axis (as is shown in Figure 3).

For example, we do a QFT on a single qubit state ∣ψ⟩=α∣0⟩+β∣1⟩|\psi \rangle = \alpha |0 \rangle + \beta |1 \rangle∣ψ⟩=α∣0⟩+β∣1⟩. Here, x0=αx_0 = \alphax0​=α, x1=βx_1 = \betax1​=β and N=2N = 2N=2. By formula (1)(1)(1), we have y0=12(α+β)y_0 = \frac{1}{\sqrt{2}} (\alpha + \beta)y0​=2​1​(α+β) and y1=12(α−β)y_1 = \frac{1}{\sqrt{2}} (\alpha - \beta)y1​=2​1​(α−β). Therefore, the final result is the state

UQFT(∣ψ⟩)=12(α+β)∣0⟩+12(α−β)∣1⟩U_{QFT}(|\psi \rangle) = \frac{1}{\sqrt{2}} (\alpha + \beta) |0\rangle + \frac{1}{\sqrt{2}} (\alpha - \beta) |1\rangle UQFT​(∣ψ⟩)=2​1​(α+β)∣0⟩+2​1​(α−β)∣1⟩

If we use the Hadamard operator H=12[111−1]H = \frac{1}{\sqrt{2}} \begin{bmatrix} 1 & 1 \\ 1 & -1\end{bmatrix}H=2​1​[11​1−1​], then UQFT(∣ψ⟩)=H∣ψ⟩U_{QFT}(|\psi \rangle) = H | \psi \rangleUQFT​(∣ψ⟩)=H∣ψ⟩.

Let’s consider a transformation for a large N=2nN = 2^nN=2n. For ∣x⟩=∣x1,x2,...,xn⟩|x \rangle = |x_1, x_2,...,x_n \rangle∣x⟩=∣x1​,x2​,...,xn​⟩, we can have the conclusion (by some algebra deduction)

QFTN(∣x⟩)=1N(∣0⟩+e2πi2x∣1⟩)⊗(∣0⟩+e2πi22x∣1⟩)⊗...⊗(∣0⟩+e2πi2nx∣1⟩)(4)QFT_N(|x\rangle) = \frac{1}{\sqrt{N}} (|0 \rangle + e^{\frac{2 \pi i}{2}x} |1 \rangle) \otimes (|0 \rangle + e^{\frac{2 \pi i}{2^2}x} |1 \rangle) \otimes ... \otimes (|0 \rangle + e^{\frac{2 \pi i}{2^n}x} |1 \rangle) \tag{4} QFTN​(∣x⟩)=N​1​(∣0⟩+e22πi​x∣1⟩)⊗(∣0⟩+e222πi​x∣1⟩)⊗...⊗(∣0⟩+e2n2πi​x∣1⟩)(4)

where ⊗\otimes⊗ means the product among different qubits. In fact, formula (4)(4)(4) describes the phenomenon in Figure 3.

Circuit

QFT circuit uses two gates: one is single-qubit Hadamard gate HHH:

H(∣xk⟩)=12(∣0⟩+e2πi2xk∣1⟩)(5)H(|x_k \rangle) = \frac{1}{\sqrt{2}} (|0 \rangle + e^{\frac{2 \pi i}{2} x_k} |1 \rangle) \tag{5} H(∣xk​⟩)=2​1​(∣0⟩+e22πi​xk​∣1⟩)(5)

and the other is a two-qubit controlled rotation RkR_kRk​ that is controlled by the kkk-th qubit:

Rk=[100e2πi2k](6)R_k = \begin{bmatrix} 1 & 0 \\ 0 & e^{\frac{2 \pi i}{2^k}}\end{bmatrix} \tag{6} Rk​=[10​0e2k2πi​​](6)

Cost

According to Figure 4, the procedure of QFTNQFT_NQFTN​ requires O(n2)=O(lg⁡2N)O(n^2) = O(\lg^2{N})O(n2)=O(lg2N) of time, since n=lg⁡Nn = \lg{N}n=lgN.

For the jjj-th qubit, it requires 1 HHH gate and n−jn-jn−j RRR gates; Therefore, the number of gates in the procedure is n+(n−1)+...+1=n(n+1)2=O(n2)n + (n-1) + ... + 1 = \frac{n(n+1)}{2} = O(n^2)n+(n−1)+...+1=2n(n+1)​=O(n2). Also, at most n2\frac{n}{2}2n​ swaps are required, and each swap can be accomplished by 3 controlled-NOT gates. Hence, QFTNQFT_NQFTN​ requires O(n2)+O(n)=O(n2)=O(lg⁡2N)O(n^2) + O(n) = O(n^2) = O(\lg^2{N})O(n2)+O(n)=O(n2)=O(lg2N) gates.

Comparison

As we mentioned before, QFTNQFT_NQFTN​ has a time and space complexity of both O(lg⁡2N)O(\lg^2{N})O(lg2N), and FFTNFFT_NFFTN​ has a time and space complexity of both O(Nlg⁡N)O(N \lg{N})O(NlgN), which requires exponentially more operations. This gap is caused by the strong expressibility of qubits, which is exponentially stronger than that of classical bits. Therefore, even if we don’t adopt a Divide&Conquer strategy for a “Fast Quantum Fourier Transform”, the speedup is still enormous.

This speedup seems terrific since Fourier Transform is common in many data processing applications. Unfortunately, there is no known way to leverage QFT to speed up this process, because it is hard to

  • determine the amplitudes of the original state ∣x1x2...xn⟩|x_1x_2...x_n \rangle∣x1​x2​...xn​⟩ and
  • access the amplitudes of the final state ∣y1y2...yn⟩|y_1y_2...y_n \rangle∣y1​y2​...yn​⟩ by measurement.
