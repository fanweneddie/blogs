Note of Data Structure Refinements

Created2022-01-15Updated2022-01-15

This is the note of paper “Synthesizing Data Structure Refinements from Integrity Constraints” by Shankara Pailoor et al. at PLDI’21.

Background

Data structures have some auxiliary fields to avoid repetitious operation and improve performance. However, the revision of those fields is not easy, and it may break the constraints and cause some potential bugs.

Problem formulation

Definition

Data Structure

A data structure(ds) is a form D=(F,M)D = (F,M)D=(F,M) where FFF is a set of fields and MMM is a mapping from method signatures to their body.

We assume that each data structure DDD has a unique method called D0D_0D0​ as a constructor. Fields and variables in DDD are either of type int(as primitive type) or Ref(D') (as reference type). The program heap H:Ref×Field→ValueH: Ref \times Field \rightarrow ValueH:Ref×Field→Value maps a pair of reference and field to values.

Data structure refinement

A ds D′D'D′ refines DDD, if

  • the set of fields of D′D'D′ is a superset of those of DDD
  • for every method mmm in DDD and its corresponding method m′m'm′ in D′D'D′, m′m'm′ refines mmm.

Integrity constraint

An integrity constraint holds on state σ\sigmaσ, denoted as σ⊨Φc\sigma \models \Phi_{c}σ⊨Φc​, if Φc\Phi_{c}Φc​ returns true on input σ\sigmaσ.

Problem

So given a ds D=(F,M)D = (F,M)D=(F,M) and an integrity constraint Φc\Phi_cΦc​, our goal is to synthesize a new ds D′=(F,M′)D' = (F,M')D′=(F,M′) such that D′≤DD' \leq DD′≤D and D′⊨ΦcD' \models \Phi_{c}D′⊨Φc​.

Theorem

Lemma 1

Suppose that UPDATEMETHOD() rejects a candidate body S′S'S′ for method mmm. Then, there does not exist a solution D′D'D′ to the synthesis problem where M′[m]=S′M'[m] = S'M′[m]=S′.

Inductive Synthesis Algorithm

The inductive synthesis algorithm is shown in Figure 5.

This algorithm uses top-down enumerative search with deduction-based pruning and grammar augmentation.

It maintains a worklist WWW of partial programs and iteratively explores partial programs until it finds a complete program that satisfies all the counterexamples.

Figure 5: SYNTHESIZE

In each iteration, it invokes SELECTBEST() function to identify the most promising partial program in the work-list.

Then, given a partial program P′P'P′, SYNTHESIZE invokes the DEDUCE function to check whether P′P'P′ is infeasible.

The high level idea is to compute a necessary precondition Φ\PhiΦ for correctness that any instantiation of PPP must satisfy and test whether Φ\PhiΦ is satisfied by the counterexamples. If any C ∈C\in C∈C violates this necessary precondition, PPP is guaranteed to be infeasible.

To be specific, the DEDUCE procedure first inlines the method calls in the PPP and calls APPROXIMATE() to generate a completion S′S'S′ of PPP. Here, S′S'S′ contains symbolic variables that represents “environment choices”.
